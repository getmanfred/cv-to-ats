Antes de empezar
----------------
- Abre la app en producción (Railway)
- Ten a mano un CV en PDF o DOCX para subir
- Ten a mano el texto de al menos 2-3 ofertas de trabajo
- Usa Chrome o Safari en escritorio + Chrome en móvil para los tests de responsive

Marca cada prueba con [OK], [FALLO] o [N/A] según el resultado.

  BLOQUE 1 — PÁGINA PRINCIPAL / ANÁLISIS DE CV

[ ] 1.1 DEMO SIN CV

  BLOQUE 3 — MATCH CON OFERTA
================================================================================

[ ] 3.1 MATCH INDIVIDUAL BÁSICO
    - Navega a /match
    - Si tienes un CV analizado reciente, debe aparecer el banner
      "CV de [nombre] cargado · Del análisis anterior"
    - Pega el texto de una oferta de trabajo y haz clic en "Comprobar match"
    - Verifica que el resultado muestra score, keywords presentes/faltantes y sugerencias

[ ] 3.2 TONO DEL RESUMEN
    - En el resultado de match, lee el párrafo de resumen
    - Si el score es bajo o medio (<75), el resumen NO debe empezar con elogios
      ("gran perfil", "impresionante trayectoria", "tienes mucho que ofrecer")
    - Debe ir directamente al análisis: ej. "Tu CV tiene un match del 45% con esta oferta..."

[ ] 3.3 MENSAJES DE CARGA TÉCNICOS
    - Durante el análisis de match, verifica que entre los mensajes de carga aparecen
      algunos técnicos: "Vectorizando keywords...", "Calculando distancia semántica CV-JD...",
      "Identificando requisitos excluyentes..."

[ ] 3.4 URL CON LOGIN (CORRECCIÓN NUEVA)
    - En el campo de oferta, pega una URL de LinkedIn Jobs o Indeed que requiera login
      (ej: https://www.linkedin.com/jobs/view/cualquier-id)
    - Debe devolver un error claro:
      "La URL requiere iniciar sesión para ver el contenido. Prueba a pegar el texto directamente."
    - NO debe intentar analizar contenido vacío ni dar un resultado sin sentido

[ ] 3.5 URL PÚBLICA (sin login)
    - Pega una URL de una oferta pública (ej: de una web de empresa directamente)
    - Debe funcionar y extraer el texto de la oferta correctamente

[ ] 3.6 BANNER ANÁLISIS ANTERIOR
    - Si ya tienes un resultado de match guardado, al entrar a /match debe aparecer
      el banner "Tienes un análisis de match anterior guardado" con botón "Ver análisis →"

[ ] 3.7 PROBAR CON OTRA OFERTA (sin duplicar URL)
    - Desde /match/results, usa el campo "Probar con otra oferta"
    - Verifica que la URL del navegador NO cambia a /match/results?... (bug anterior)
    - Debe quedarse en /match/results y actualizar el contenido

[ ] 3.8 LINK AL MODO BATCH
    - En /match, debe aparecer el texto "¿Tienes varias ofertas? Compara todas a la vez →"
    - Haz clic → debe navegar a /match/batch

[ ] 3.9 MATCH EN BATCH (IDEA-06)
    - Navega a /match/batch
    - Comprueba que aparecen 2 textareas por defecto
    - Haz clic en "Añadir oferta" → debe aparecer una tercera (hasta máximo 8)
    - Pega texto de 2-3 ofertas distintas (mín. 50 caracteres cada una)
    - Opcionalmente añade etiquetas a cada oferta
    - Haz clic en "Analizar X ofertas"
    - Debe aparecer una barra de progreso: "Analizando oferta 1 de 3...", "2 de 3..."
    - Al terminar: ranking ordenado por score, con keywords verdes (match) y rojas (gap)
    - Haz clic en "Ver análisis completo →" de la oferta con mejor score
    - Debe navegar a /match/results con ese resultado

================================================================================
  BLOQUE 9 — CALIDAD DEL ANALISIS (PR #9)
================================================================================

Estas pruebas verifican las mejoras de consistencia y precision del analisis ATS.
Usa un CV real tuyo para las tres pruebas seguidas, sin cerrar el navegador entre ellas.

[ ] 9.1 CONSISTENCIA DE PUNTUACION (varianza)
    - Sube el mismo CV tres veces seguidas sin cerrar la pestaña
    - Anota la puntuacion global de cada analisis
    - Las tres puntuaciones deben estar dentro de un rango de ±3 puntos
    - Si la diferencia es >5 puntos, hay varianza inaceptable — reportar
    - Hace lo mismo con una categoria concreta (ej: "Keywords & skills"):
      las tres puntuaciones de esa categoria deben estar dentro de ±5 puntos

[ ] 9.2 SIN CONSEJOS DE FECHAS
    - Analiza un CV que tenga experiencia de 2024 y/o 2025
    - Revisa todas las sugerencias generadas
    - NINGUNA sugerencia debe recomendar:
      * "Actualiza las fechas de experiencia"
      * "Corrige el año de [empresa]"
      * "La fecha de [rol] parece estar en el futuro"
      * Cualquier variante de cambiar o corregir fechas temporales
    - Las fechas 2024-2026 deben tratarse como hechos validos del candidato

[ ] 9.3 CV ADAPTADO NO EMPEORA
    - Analiza el CV original y anota la puntuacion (ej: 68)
    - Aplica al menos 2-3 de las sugerencias de alta prioridad al CV
      (puedes usar el editor o editar directamente el archivo)
    - Sube el CV mejorado y anota la nueva puntuacion
    - La nueva puntuacion debe ser >= a la original, o a lo sumo 3 puntos menor
    - Si el CV mejorado baja más de 5 puntos, es un fallo — reportar con detalle
      (que sugerencias aplicaste, cuanto bajo, que categorias cambiaron)

[ ] 9.4 FORMATO LIMPIO NO SE PENALIZA
    - Crea un CV en el editor Harvard y descargalo como PDF o DOCX
    - Subelo al analizador
    - La categoria "Formato y parseo" (o "Format & parseability") debe puntuar
      al menos 70/100 — el formato Harvard es limpio y single-column
    - Si puntua menos de 60, hay un problema en los criterios de scoring — reportar
